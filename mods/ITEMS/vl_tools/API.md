# `vl_tools` API

`vl_tools` provides tool-type namespaces for registering tools and reusing their related callbacks.

## Common Fields

Tool registration definitions use structured Luanti-style subtables where practical. Tool-type APIs add their default values first, then merge the provided subtables over those defaults.

### Item Fields

* `description`: Display name for the registered tool.
* `icon`: Inventory image texture. Tool-type APIs may require this field.
* `repair_material`: Item name or group used to repair the tool. Tool-type APIs may also use this for crafting recipes.
* `_doc_items_hidden`: Optional doc item visibility override.
* `_mcl_upgradable`: Marks a tool as upgradable.
* `_mcl_upgrade_item`: Item name of the upgraded tool.

### Durability and Mining Fields

* `_mcl_diggroups`: Dig group capability table used by `mcl_autogroup`.
* `uses`: Optional shorthand for both punch uses and default dig uses.
* `dig_uses`: Optional shorthand for dig durability. When omitted, tool APIs may use `uses`.
* `speed`: Optional shorthand for the tool's primary dig group speed.
* `level`: Optional shorthand for the tool's primary dig group harvest level.

### Combat Fields

* `tool_capabilities`: Luanti tool capability table.
* `damage`: Optional shorthand for `tool_capabilities.damage_groups.fleshy`.
* `full_punch_interval`: Optional shorthand for `tool_capabilities.full_punch_interval`.
* `max_drop_level`: Optional shorthand for `tool_capabilities.max_drop_level`.

### Enchanting and Groups

* `groups`: Extra or overriding item groups. Tool APIs merge this into their default group table.
* `enchantability`: Optional shorthand for `groups.enchantability`.
* `dig_speed_class`: Optional shorthand for `groups.dig_speed_class`.

### Crafting Fields

* `no_craft`: If true, suppresses default crafting recipe registration.
* `fuel_burntime`: If set, registers the tool as fuel with this burn time.
* `cook_result`: If set, registers a cooking recipe from the tool to this output.
* `cooktime`: Optional cooking time. Defaults to 10 where supported.

### Override Fields

* `overrides`: Extra or overriding fields applied directly to the final registered tool definition.

Use `overrides` for fields that are not part of a tool-type API yet.

## Axe API

### `vl_tools.axe.register(name, def)`

Registers an axe and its default crafting-related recipes.

Required fields:

* `name`: Full item name passed as the first argument.
* `def.icon`
* `def.repair_material`

The function asserts that all required fields are present.

Definition layout:

* Required: Basic identity fields needed to register the tool.
* Groups: Item groups used by creative inventory, enchanting, capabilities, and special behavior.
* Tool capabilities: Luanti `tool_capabilities` values for combat and drop behavior.
* Dig groups: `_mcl_diggroups` values used by `mcl_autogroup`.
* Crafting: Recipe-related options generated by the registration helper.
* Upgrades: Smithing or upgrade metadata.
* Documentation: Doc visibility or text overrides.
* Overrides: Direct final-definition overrides for fields not covered by the API yet.

Default behavior:

* Registers the item with `core.register_tool`.
* Adds `tool = 1` and `axe = 1` groups.
* Sets `_mcl_toollike_wield = true`.
* Sets `_repair_material = def.repair_material`.
* Uses `vl_tools.axe.make_stripped_trunk` as `on_place`.
* Merges `groups`, `tool_capabilities`, and `_mcl_diggroups` into axe defaults.
* Creates `_mcl_diggroups.axey` from `speed`, `level`, and `dig_uses or uses` when `_mcl_diggroups.axey` is not provided.
* Creates mirrored axe crafting recipes from `repair_material`, unless `no_craft` is true.
* Registers optional fuel and cooking recipes from `fuel_burntime` and `cook_result`.

Example:

```lua
vl_tools.axe.register("example:axe_copper", {
	description = "Copper Axe",
	icon = "example_axe_copper.png",
	repair_material = "example:copper_ingot",

	groups = {
		dig_speed_class = 4,
		enchantability = 14,
	},

	tool_capabilities = {
		full_punch_interval = 1.1,
		max_drop_level = 4,
		damage_groups = { fleshy = 8 },
		punch_attack_uses = 190,
	},

	_mcl_diggroups = {
		axey = { speed = 5, level = 4, uses = 190 },
	},

	cook_result = "example:copper_nugget",
})
```

### `vl_tools.axe.make_stripped_trunk(itemstack, placer, pointed_thing)`

Default axe `on_place` callback.

It handles:

* Calling a pointed node's right-click handler when appropriate.
* Protection checks.
* Replacing nodes that define `_mcl_stripped_variant`.
* Unlocking the wax-off achievement for waxed nodes.
* Applying axe wear through `mcl_autogroup.get_wear`.
